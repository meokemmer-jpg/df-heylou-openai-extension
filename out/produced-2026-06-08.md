# df-heylou-openai-extension: Primäres Output-Artefakt

## Architekturbeschreibung

Die Dark-Factory `df-heylou-openai-extension` integriert die HeyLou-Funktionalitäten in das OpenAI Function-Calling Muster, um eine vollständige und sichere Integration für externe Anwendungen zu ermöglichen. Das System besteht aus mehreren wichtigen Modulen:

### Modul: `openai_extension.py`

Dieses Modul enthält die Definition der Funktionen, die von den Anwendern aufgerufen werden können und direkt zu den entsprechenden Backend-Modulen weitergeleitet werden:
```python
import json

def handle_function_call(function_call):
    function_name = function_call['name']
    if function_name == 'search_hotels':
        return search_hotels(**function_call['arguments'])
    elif function_name == 'get_rates':
        return get_rates(**function_call['arguments'])
    # ähnliche Logik für die anderen Funktionen
```

### Modul: `auth_handler.py`

Dieses Modul verarbeitet Authentifizierungs- und Berechtigungsanfragen, um sicherzustellen, dass nur berechtigte Benutzer auf die API-Zugriff haben:
```python
import os

def verify_phronesis_ticket(ticket):
    if ticket == os.getenv('PHRONESIS_TICKET'):
        return True
    else:
        return False
```

### Modul: `audit_logger.py`

Dieses Modul speichert jede Funktionseintrag in einer JSONL-Datei für Audit und Sicherheitszwecke:
```python
import json
from datetime import datetime

def log_audit(function_call, result):
    audit_entry = {
        'function_name': function_call['name'],
        'arguments': function_call.get('arguments', {}),
        'result': result,
        'timestamp': datetime.now().isoformat()
    }
    with open('audit_log.jsonl', 'a') as file:
        json.dump(audit_entry, file)
        file.write('\n')
```

### Modul: `adapter_orchestrator.py`

Dieses Modul ist das Eingangspunkt für die Dark-Factory und startet den LaunchAgent:
```python
if __name__ == "__main__":

```

## Sandbox-Default

Die Sandbox-Einstellungen sind auf zwei mögliche Szenarien angepasst:

1. **Mock-Responses (HeyLou Travel-Knowledge-Graph synthetisch)**
   - `DF_HEYLOU_OPENAI_EXT_ENABLED=false`: Wenn diese Umgebungsvariable nicht gesetzt ist oder den Wert `false` hat, werden Mock-Antworten generiert.
   
2. **Real-Mode**
   - `DF_HEYLOU_OPENAI_EXT_ENABLED=true`: Wenn diese Umgebungsvariable den Wert `true` hat und zusätzlich die Umgebungsvariablen `PHRONESIS_TICKET` und `OPENAI_API_KEY` gesetzt sind, wird der Realmodus aktiviert.

## Cross-DF-Coupling (W36/W37 Backends)

Die Dark-Factory ist so konzipiert, dass sie mit anderen DFs wie `df-heylou-travel-domain`, `df-pms-mews-adapter` und `df-ota-*` kommunizieren kann. Aktuell sind diese Interaktionen in Form von Lazy Import Stubs implementiert.

### Beispiel für eine Lazy Import-Funktion
```python
def get_rates(hotel_id, date_range):
    from df_pms_mews_adapter import PMSAdapter  # Lazy Import
    pms = PMSAdapter()
    rates = pms.fetch_rates(hotel_id, date_range)
    return rates
```

## K11-K16 + LC1-LC5

Die Implementierung der Kompromissanalyse (K13) sorgt dafür, dass jede Funktion nur ausgeführt wird, wenn ein gültiges `PHRONESIS_TICKET` bereitgestellt wurde. Dies geschieht durch die Verwendung des Moduls `auth_handler.py`.

### Beispiel für eine Pre-Action-Verification
```python
def pre_action_verification():
    ticket = os.getenv('PHRONESIS_TICKET')
    if not auth_handler.verify_phronesis_ticket(ticket):
        raise PermissionError("Invalid or missing PHRONESIS_TICKET")
```

## Implementierung der Funktionen

### search_hotels(location, dates, preferences)

Diese Funktion ermöglicht es Benutzern, Hotels in einem bestimmten Gebiet für gegebene Daten aufgrund ihrer Präferenzen zu durchsuchen. Der Aufruf wird an den `df-heylou-travel-domain`-Backend weitergeleitet.
```python
def search_hotels(location, dates, preferences):
    if not verify_phronesis_ticket(os.getenv('PHRONESIS_TICKET')):
        return {"error": "Unauthorized access"}
    
    response = handle_function_call({
        'name': 'search_hotels',
        'arguments': {
            'location': location,
            'dates': dates,
            'preferences': preferences
        }
    })
    
    log_audit(function_call={'name': 'search_hotels', 'arguments': {'location': location, 'dates': dates, 'preferences': preferences}}, result=response)
    return response
```
